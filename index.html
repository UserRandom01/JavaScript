JavaScript

Introduction

JavaScript is a versatile, high-level programming language that is primarily used for enhancing the interactivity and functionality of websites. Initially developed by Netscape in 1995, it has become a cornerstone of modern web development, enabling dynamic content updates, form validation, interactive animations, and complex user interfaces without requiring page reloads. JavaScript operates on the client side, meaning it runs directly within the web browser, which allows for immediate feedback and seamless user experiences. Its ecosystem is vast, supported by numerous libraries and frameworks like React, Angular, and Vue.js, which facilitate the creation of sophisticated web applications. Beyond the browser, JavaScript has extended its reach through environments like Node.js, enabling server-side programming and the development of full-stack applications. With its ease of learning and widespread adoption, JavaScript continues to be a fundamental skill for web developers.

Client-Side Language
JavaScript is often used on the client side (i.e., in the browser) to add interactivity to web pages. It can respond to user actions like clicks, form submissions, and other events.
Server-Side Use
JavaScript can also be used on the server side, thanks to environments like Node.js, which allows developers to use JavaScript to write server-side code.
Scripting Language
It's a lightweight scripting language, which means it's designed to be executed within another program, like a web browser.
Versatile and Dynamic
JavaScript can be used to manipulate HTML and CSS, control multimedia, animate elements, and more.
Event-Driven
JavaScript can respond to events such as user inputs, time-based events, and others, making it ideal for creating dynamic web applications.
Object-Oriented
JavaScript supports object-oriented programming, which means you can create objects to model real-world entities and manage complex data.

Alert Function

The alert function in JavaScript is a simple way to display a message to the user in a dialog box. It is typically used for debugging purposes or to show important information that the user needs to acknowledge before continuing.

alert("Hello");

When this code is executed, a dialog box will appear with the message "This is an alert message!" and an "OK" button. The script execution will pause until the user dismisses the alert by clicking the "OK" button.

Console

In JavaScript, the console object provides access to the browser's debugging console. It is a tool commonly used for logging information, debugging code, and testing. The console object has several methods, with console.log() being one of the most frequently used.

The console.log() method is used to output messages to the web console. It is often used for debugging purposes, allowing developers to print variables, objects, and other information to understand the state and flow of the program.

You can use console.log() to log the values of variables and objects:
let name = "Sameer";
let age = 21;
let user = { name: "Bob", age: 25 };

console.log(name); // Outputs: Alice
console.log(age); // Outputs: 30
console.log(user); // Outputs: { name: "Bob", age: 25 }
Common console Methods
  • console.log(message): Logs a message to the console.
  • console.error(message): Logs an error message.
  • console.warn(message): Logs a warning message.
  • console.info(message): Logs an informational message.
  • console.table(data): Displays tabular data as a table.
  • console.time(label): Starts a timer with a specific labe

The console object and its methods, especially console.log(), are invaluable tools for debugging and understanding the behavior of your code. They provide a straightforward way to track variables, inspect objects, measure execution time, and more, all from within the development environment.

Variables & Data Types

In JavaScript, variables are used to store data that can be referenced and manipulated within a program. They act as containers for data values. Variables can store various types of data, including numbers, strings, objects, arrays, functions, and more.

Variable Rules

  • Variable names are case sensitive; “a” & “A” is different.
  • Only letters, digits, underscore( _ ) and $ is allowed. (not even space)
  • Only a letter, underscore( _ ) or $ should be 1st character.
  • Reserved words cannot be variable names.

In JavaScript, you can declare variables using the var, let, or const keywords:

  • var: The var keyword is used to declare a variable. It has function scope and can be redeclared.
    var name = "Alice";
    console.log(name); Outputs: Alice

    var age = 30;
    console.log(age); Outputs: 30
  • let: The let keyword is used to declare a block-scoped variable. It cannot be redeclared within the same scope.
    let city = "New York";
    console.log(city); Outputs: New York

    city = "Los Angeles";
    console.log(city); Outputs: Los Angeles
  • const: The const keyword is used to declare a block-scoped, read-only constant. The value of a const variable cannot be changed through reassignment.
    const country = "USA";
    console.log(country); Outputs: USA

    Attempting to reassign a const variable will result in an error
    // country = "Canada"; // Uncaught TypeError: Assignment to constant variable.

Varibales Scope

The scope of a variable determines where it can be accessed within the code.

  • Global Scope: Variables declared outside of any function or block have global scope. They can be accessed from anywhere in the code.
    var globalVar = " I am global" ;
    function test() {
    console.log(globalVar); Outputs: I am global
    }
    test();
    console.log(globalVar); Outputs: I am global
  • Function Scope: Variables declared with var inside a function are scoped to that function. They cannot be accessed from outside the function.
    function test() {
    var localVar = "I am local";
    console.log(localVar); Outputs: I am local
    }

    test();
    console.log(localVar); // Uncaught ReferenceError: localVar is not defined
  • Block Scope: Variables declared with let or const inside a block (e.g., inside {}) are scoped to that block.
    {
    let blockVar = "I am block scoped";
    console.log(blockVar); Outputs: I am block scoped
    }
    console.log(blockVar); // Uncaught ReferenceError: blockVar is not defined

var:Variable can be re-declared and updated.A global scope variable.

let:Variable cannot be re-declared but can be updated.A block scope variable.

const:Variable cannot be re-declared or updated.A block scope variable.

Conclusion:

Variables are fundamental to programming in JavaScript. They are used to store and manipulate data, and understanding their behavior, scope, and the differences between var, let, and const is crucial for writing effective and efficient JavaScript code.

Operators

Operators in JavaScript are symbols or keywords used to perform operations on values or variables. They help in manipulating data and making decisions based on conditions. JavaScript has several types of operators, each serving different purposes.

There are primarily 7 types of operators in JavaScript,they are:

  • Arithmetic Operators
  • Assignment Operators
  • Comparison Operators
  • Logical Operators
  • Conditional (Ternary) Operator
  • Bitwise Operators
  • String Operators

Arithmetic Operators

Arithmetic operators in JavaScript are used to perform basic mathematical operations. Here's a quick overview of each one:

Addition (+)

Adds two numbers.

let sum = 5 + 3;  // sum = 8

Subtraction (-)

Subtracts one number from another.

let difference = 10 - 4;  // difference = 6

Multiplication (*)

Multiplies two numbers.

let product = 6 * 7;  // product = 42

Division (/)

Divides one number by another.

let quotient = 20 / 5;  // quotient = 4

Modulus (%)

Returns the remainder of a division operation.

let remainder = 10 % 3;  // remainder = 1

Exponentiation (**)

Raises a number to the power of another number.

let power = 2 ** 3;  // power = 8 (2^3)

Increment (++)

Increases a number by 1.

let count = 5;
                    count++;  // count = 6

Decrement (--)

Decreases a number by 1.

let count = 5;
                    count--;  // count = 4

Assignment Operators

Assignment operators in JavaScript are used to assign values to variables. They can also perform operations and then assign the result to a variable. Here's an overview of the different assignment operators:

Assignment (`=`)

Assigns the right-hand operand's value to the left-hand operand.

let x = 10;  // x = 10

Addition Assignment (`+=`)

Adds the right-hand operand to the left-hand operand and assigns the result to the left-hand operand.

let x = 5;
x += 3;  // x = 8

Subtraction Assignment (`-=`)

Subtracts the right-hand operand from the left-hand operand and assigns the result to the left-hand operand.

let x = 10;
x -= 4;  // x = 6

Multiplication Assignment (`*=`)

Multiplies the left-hand operand by the right-hand operand and assigns the result to the left-hand operand.

let x = 6;
x *= 2;  // x = 12

Division Assignment (`/=`)

Divides the left-hand operand by the right-hand operand and assigns the result to the left-hand operand.

let x = 20;
x /= 5;  // x = 4

Modulus Assignment (`%=`)

Takes the modulus of the left-hand operand by the right-hand operand and assigns the result to the left-hand operand.

let x = 10;
x %= 3;  // x = 1

Exponentiation Assignment (`**=`)

Raises the left-hand operand to the power of the right-hand operand and assigns the result to the left-hand operand.

let x = 2;
x **= 3;  // x = 8

Comparison Operators

Comparison operators in JavaScript are used to compare two values and return a boolean result (true or false). Here’s a breakdown of each comparison operator:

Equal to (`==`)

Compares two values for equality, performing type coercion if necessary.

let a = 5;
let b = '5';
console.log(a == b);  // true

Strict Equal to (`===`)

Compares two values for equality without performing type coercion. Both the value and type must be the same.

let a = 5;
let b = '5';
console.log(a === b);  // false

Not Equal to (`!=`)

Compares two values for inequality, performing type coercion if necessary.

let a = 5;
let b = '6';
console.log(a != b);  // true

Strict Not Equal to (`!==`)

Compares two values for inequality without performing type coercion. Both the value and type must be different.

let a = 5;
let b = '5';
console.log(a !== b);  // true

Greater Than (`>`)

Checks if the left-hand operand is greater than the right-hand operand.

let a = 10;
let b = 5;
console.log(a > b);  // true

Greater Than or Equal to (`>=`)

Checks if the left-hand operand is greater than or equal to the right-hand operand.

let a = 10;
let b = 10;
console.log(a >= b);  // true

Less Than (`<`)< /h4>

Checks if the left-hand operand is less than the right-hand operand.

let a = 5;
let b = 10;
console.log(a < b);  // true

Less Than or Equal to (`<=`)< /h4>

Checks if the left-hand operand is less than or equal to the right-hand operand.

let a = 5;
let b = 10;
console.log(a <= b);  // true

Example Combining Comparison Operators

let a = 10;
let b = 5;

console.log(a == 10);   // true
console.log(a === '10');  // false
console.log(a != b);    // true
console.log(a !== 5);   // false
console.log(a > b);     // true
console.log(a >= 10);   // true
console.log(a < b);     // false
console.log(a <= 10);   // true

Logical Operators

Logical operators in JavaScript are used to combine multiple boolean expressions and return a boolean result. Here’s a breakdown of each logical operator:

Logical AND (`&&`)

Returns true if both operands are true. If the first operand is false, it returns false without evaluating the second operand (short-circuit evaluation). 

let a = true;
let b = false;
console.log(a && b);  // false

Logical OR (`||`)

Returns true if at least one of the operands is true. If the first operand is true, it returns true without evaluating the second operand (short-circuit evaluation).

let a = true;
let b = false;
console.log(a || b);  // true

Logical NOT (`!`)

Returns true if the operand is false, and false if the operand is true. It negates the boolean value of the operand.

let a = true;
console.log(!a);  // false

Logical AND with Assignment (`&&=`)

Logical AND combined with assignment. Assigns the right-hand operand to the left-hand operand only if the left-hand operand is true.

let a = true;
let b = false;
a &&= b;
console.log(a);  // false

Logical OR with Assignment (`||=`)

Logical OR combined with assignment. Assigns the right-hand operand to the left-hand operand only if the left-hand operand is false.

let a = false;
let b = true;
a ||= b;
console.log(a);  // true

Logical Nullish Coalescing with Assignment (`??=`)

Logical nullish coalescing combined with assignment. Assigns the right-hand operand to the left-hand operand only if the left-hand operand is null or undefined. 

let a = null;
let b = 'default';
a ??= b;
console.log(a);  // 'default'

Example Combining Logical Operators

let a = true;
let b = false;
let c = null;

console.log(a && b);  // false
console.log(a || b);  // true
console.log(!a);      // false
console.log(a &&= b); // false
console.log(c ||= 'default');  // 'default'
console.log(c ??= 'new default');  // 'default' (unchanged)

Ternary Operators

The ternary operator in JavaScript is a conditional operator that provides a shorthand way to perform conditional logic. It is the only operator in JavaScript that takes three operands, hence the name "ternary." Here's how it works:

Ternary Operator (`? :`)

The ternary operator is a shorthand for the if-else statement. It evaluates a condition and returns one of two values based on whether the condition is true or false.

let age = 18;
let isAdult = (age >= 18) ? 'Yes, adult' : 'No, not an adult';
console.log(isAdult);  // 'Yes, adult'

Nesting Ternary Operators

You can nest ternary operators to handle more complex conditions. Each ternary operator is evaluated in order.

let score = 85;
let grade = (score >= 90) ? 'A' :
            (score >= 80) ? 'B' :
            (score >= 70) ? 'C' : 'F';
console.log(grade);  // 'B'

Example Combining Ternary Operators

Combining multiple ternary operators can help simplify conditional logic in some cases.

let a = 10;
let b = 20;
let result = (a > b) ? 'a is greater than b' :
            (a < b) ? 'a is less than b' : 'a is equal to b';
console.log(result);  // 'a is less than b'
